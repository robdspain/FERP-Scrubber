About FERPA Scrubber

How your data is handled

  • On De-identify, the Edge Function replaces sensitive items with tokens (e.g., [[FERPA:EMAIL:1]]).
  • Original values are encrypted on the Edge using AES‑256‑GCM; the key is returned to your browser.
  • AI features only receive the de‑identified text — encrypted values and keys are never sent to the LLM.
  • Decryption occurs on the Edge only when you click Decrypt, using the key and map held in your browser memory.
  • No sensitive plaintext is stored server-side.

What you control

  • Select which items to protect (emails, student emails, phones, SSNs, addresses, names, student IDs, dates).
  • Optionally switch themes for readability.
